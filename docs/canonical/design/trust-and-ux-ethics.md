# HAVI Trust And UX Ethics

## Purpose

Define non-negotiable UX trust rules for launch-critical product surfaces.

## Anti-Dark-Pattern Rules

1. Do not show fabricated usage counts, adoption stats, or activity metadata.
2. Do not imply feature availability when a feature is preview-only.
3. Disabled/preview states must be explicit and non-deceptive.
4. Error and lock states must explain what is blocked and the shortest path forward.

## Profile Lock Policy

1. Profile lock is fully blocking until required profile data is complete.
2. Blocking copy must clearly list why it is required and what is missing.
3. CTA should move users directly to completion flow.
4. Do not hide lock behavior behind ambiguous failures.

## Copy Integrity

1. Claims must be true, specific, and attributable.
2. Placeholder copy can exist on non-priority marketing pages but must not fake outcomes or customer proof.
3. Preview language should prefer: `Preview`, `Not live yet`, `Coming soon`.

## Visual Integrity

1. Respect tokenized color and contrast rules.
2. Maintain calm, restrained hierarchy; avoid urgency styling for normal actions.
3. Use consistent notice styles (`info`, `warning`, `danger`) for user trust and predictability.
