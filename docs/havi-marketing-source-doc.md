## “Why Havi” — 6 Pain Points → Solutions (scannable)

### Built for real families. Designed for the mental load.

**1) Too much to remember**  
Details slip—not because they don’t matter, but because there are too many of them.  
**Havi** keeps moments, notes, and questions in one place, so nothing valuable disappears.

**2) Advice everywhere. Clarity nowhere.**  
Parents are surrounded by information, often conflicting, rarely contextual.  
**Havi** helps you make sense of guidance in the context of *your* child and *your* situation.

**3) Sleep deprivation changes everything.**  
When sleep is broken, memory and decision-making suffer.  
**Havi** remembers for you—so you don’t have to rely on recall when you’re exhausted.

**4) Care is shared, but information isn’t.**  
Partners, grandparents, and caregivers all need alignment. Text threads don’t scale.  
**Havi** keeps everyone on the same page, without more messages or meetings.

**5) Parenting evolves constantly**  
What matters this month won’t matter next month.  
**Havi** adapts over time, surfacing what’s relevant now—not everything at once.

**6) Most tools add work**  
Logging, tagging, organizing—it all becomes another task.  
**Havi** is designed to fit into real life, capturing information naturally and using it only when it’s actually helpful.

## Competitive Grid — Young Parent Tools Compared with Havi (Bain-style)

### Executive takeaway
Most parent tools optimize **tracking**. Most general tools optimize **notes/tasks**.  
Havi’s wedge is: **memory + conversation + foresight (Runway)** across the whole family—without turning parenting into data entry.

### Comparison table (8 competitors)
| Tool / Category | Primary job-to-be-done | Strengths | Where it breaks for parents | Havi advantage |
|---|---|---|---|---|
| **Huckleberry (sleep / routines)** | Manage sleep schedules & predictions | Strong routine views; sleep focus | Becomes “one more app” for data entry; narrow scope outside sleep | Havi reduces logging via conversation; broader family memory + decisions |
| **Nara Baby (tracking)** | Log feeds/diapers/sleep | Simple tracking UI | High logging burden; limited meaning/search; weak narrative of growth | Havi learns from moments and questions; builds searchable family memory |
| **Baby Connect / generic trackers** | Comprehensive baby tracking | Lots of categories | Complexity grows; insights feel generic; coordination is hard | Havi focuses on clarity + context; shared family brain |
| **Apple Notes / Google Keep** | Capture notes quickly | Ubiquitous; fast | Unstructured; hard to retrieve at the right moment; no “what’s next” | Havi is purpose-built retrieval + Runway + parenting context |
| **Notion (workspace)** | Organize information & docs | Powerful structure | Setup overhead; too much ceremony; not designed for sleep-deprived moments | Havi is lightweight, conversational, and parent-native |
| **Google Calendar / reminders** | Schedule and remind | Reliable reminders | Doesn’t capture context or history; “why” gets lost | Havi couples reminders with memory + context + guidance |
| **WhatsApp / iMessage threads** | Coordinate caregivers | Fast communication | Decisions get buried; info fragments; hard to onboard new caregiver | Havi becomes the shared source of truth, not another thread |
| **Cozi / family organizer** | Family schedules & lists | Household coordination | Limited intelligence/context; doesn’t adapt to child development or questions | Havi combines coordination + understanding + foresight |

### Positioning summary (Bain-style bullets)
- **Category reality:** Parents stitch together point solutions (tracker + notes + calendar + texts).  
- **Core gap:** No single “family brain” that remembers + explains + anticipates.  
- **Havi wedge:** Conversation-first capture + durable memory + Runway foresight.  
- **Result:** Less cognitive load, fewer dropped balls, calmer decisions.

## Sequoia Capital Pitch Memo — Havi (updated)

### Company purpose (one sentence)
Havi is a family brain that learns as you live—helping parents reduce mental load and close the gap to the parent they want to be.

### Problem
Parenting is cognitively and emotionally demanding:
- Information is fragmented (apps, notes, texts, calendars, advice feeds).
- Sleep deprivation makes recall and decision-making harder.
- Care is shared across multiple people, but context isn’t.
- Most tools require logging; parents don’t have time for more systems.

### Solution
A calm, conversational product that:
- Captures family context naturally (chat/voice + lightweight moments).
- Remembers and retrieves what matters at the right time.
- Guides parents with a clear path forward via **Runway** (what’s ahead, what matters now).
- Works across caregivers while preventing cross-family data leakage.

### Why now
- Parents are more digitally coordinated than ever, yet more overloaded.
- Voice + chat interfaces are now mainstream and fast enough for “in the moment” support.
- Vector search + retrieval enables “family memory” that feels magical without heavy logging.
- There’s rising willingness to pay for tools that reduce stress and improve family coordination (especially in the first 3 years).

### Market potential
Initial ICP:
- New parents (0–3 years) in high-intent “overwhelm” windows
- Families with multiple caregivers (partner + nanny/daycare + grandparents)
Expansion:
- Multi-child families, broader household operations, school-age coordination
Market shape:
- Starts as consumer subscription; expands into family plans and partner ecosystem.

### Competition / alternatives
Alternatives include baby trackers, general note apps, calendars, and group chats.
- Trackers: narrow, logging-heavy
- Notes/calendars: generic, hard to retrieve contextually
- Chats: fast, but fragmented and non-searchable
Havi wins by being purpose-built for parenting mental load: **memory + conversation + foresight**.

### Business model
- Freemium (free sign-up)
- Subscription: monthly / annual
- Family plans (up to 5 users)
- Longer-term: partner channels (pediatric practices, doulas, sleep consultants), affiliate/resource marketplace (optional, later)

### Team
Built by a product-led team with strength in:
- consumer UX that feels premium and calm
- data + retrieval systems (family memory)
- operational excellence (onboarding, funnels, activation loops)

### Go-to-market
- Marketing site → focused entry points by pain: sleep deprivation, coordination, “what’s next”
- Stories: real parent moments and outcomes
- Referrals: care team invites (partners/caregivers) as built-in growth loop
- Activation metric: user experiences “another brain” moment (retrieval + Runway clarity), not just logging

### Metrics that matter (north star + funnel)
North star (proposed):
- **Weekly “confidence moments”** (times a user gets the right info/guidance at the right time)
Activation (examples):
- User asks a question and Havi answers using their saved context
- User sets up child profile + sees Runway milestone preview
- User adds a second caregiver and shares alignment
Funnel instrumentation:
- Marketing landing → signup start → wizard step completion → activation event → subscription consideration

### Vision (5 years)
Havi becomes the default operating system for modern families:
- the trusted memory of what happened
- the guide for what’s next
- the calm interface families use to coordinate, decide, and grow together
